# 📱 アプリケーション

## イントロダクション

Logto for OIDCでは、_アプリケーション_とはLogtoプラットフォームに登録され、ユーザー情報へのアクセス権限を付与されたり、ユーザーに代わってアクションを実行する権限が付与された特定のソフトウェアプログラムまたはサービスを指します。アプリケーションは、Logto APIへのリクエスト元を識別するために使用され、アプリケーションにアクセスしているユーザーの認証および認可プロセスを管理するためにも使用されます。

ログとのサインイン体験におけるアプリケーションの使用は、ユーザーが一元で自分の認可されたアプリケーションに簡単にアクセスして管理し、一貫性があり、安全な認証プロセスを提供することで、ユーザー体験を簡素化し、機密情報にアクセスしたり、組織を代表してアクションを実行する権限がある個人だけがアクセスできるようにします。

アプリケーションは、ログとの監査ログにおいても使用され、ユーザーのアクティビティを追跡し、潜在的なセキュリティの脅威や侵害を識別します。特定のアクションを特定のアプリケーションに関連付けて、Logtoはデータがどのようにアクセスされ、使用されているかについて詳細な洞察を提供することができ、組織がセキュリティおよびコンプライアンス要件をより適切に管理することができます。
Logtoとのアプリケーション統合については、[Logtoをアプリケーションに統合する](../../recipes/integrate-logto)を参照してください。

### アプリケーション名

_アプリケーション名_は、アプリケーションの人間が読み取り可能な名称であり、管理コンソールに表示されます。

_アプリケーション名_はLogtoでのアプリケーション管理の重要な要素であり、管理者がプラットフォーム内の個々のアプリケーションの活動を簡単に識別し追跡することを可能にします。

```
_アプリケーション名_は慎重に選ばれるべき点が重要であり、管理コンソールへのアクセス権を持つ全ユーザーに見えるようになります。それはアプリケーションの目的と機能を正確に反映し、理解しやすく認識しやすいものであるべきです。
```

### 説明

アプリケーションの簡単な説明が管理コンソールのアプリケーション詳細ページに表示されます。説明は、アプリケーションの目的、機能、およびその他の関連する詳細情報について管理者に追加の情報を提供することを意図しています。

### 認証エンドポイント

_認証エンドポイント_はOIDCの用語であり、ユーザーの認証プロセスを開始するために使用される必要なエンドポイントです。ユーザーがLogtoプラットフォームに登録された保護されたリソースやアプリケーションにアクセスしようとすると、_認証エンドポイント_にリダイレクトされ、リクエストされたリソースへのアクセスの認可を取得して身元を認証します。

[認証エンドポイント](https://openid.net/specs/openid-connect-core-1_0.html#AuthorizationEndpoint)で詳細情報を確認できます。

### リダイレクト URI

_リダイレクト URI_は、アプリケーション用に事前に設定された有効なリダイレクトURIのリストです。ユーザーがLogtoにサインインしてアプリケーションにアクセスしようとすると、アプリケーションの設定で指定された許可されたURIのいずれかにリダイレクトされます。

許可されたURIのリストは、アプリケーションが認証プロセス中にLogtoに送信する認証リクエストに含まれるリダイレクトURIを検証するために使用されます。認証リクエストに指定されたリダイレクトURIがアプリケーション設定の許可されたURIのいずれかと一致する場合、ユーザーは認証に成功した後、そのURIにリダイレクトされます。リダイレクトURIが許可リストにない場合、ユーザーはリダイレクトされず、認証プロセスは失敗します。

```
ユーザーが認証後にアプリケーションにうまくアクセスできるように、有効なリダイレクトURIをすべてLogtoのアプリケーションの許可リストに追加することが重要です。
```

[リダイレクションエンドポイント](https://datatracker.ietf.org/doc/html/rfc6749#section-3.1.2)で詳細情報を確認できます。

### サインアウト後のリダイレクト URI

_サインアウト後のリダイレクト URI_は、ユーザーがLogtoからサインアウトした後にユーザーをリダイレクトするために事前に設定された有効なURIのリストです。

ログアウトのための許可された_サインアウト後のリダイレクト URI_の使用は、OIDCのRP-Initiated Logoutの仕様の一部です。この仕様はアプリケーションがユーザーのためのログアウトリクエストを開始するための標準化された方法を提供し、サインアウトした後にユーザーを事前に設定されたエンドポイントにリダイレクトします。

ユーザーがLogtoからサインアウトすると、そのセッションは終了し、アプリケーション設定で指定された許可されたURIのいずれかにリダイレクトされます。これにより、ユーザーがサインアウトした後に承認された有効なエンドポイントのみにリダイレクトされることが保証され、未知または未確認のエンドポイントにユーザーをリダイレクトした際に生じる不正アクセスやセキュリティリスクを防ぐことに役立ちます。

[RP-Initiated Logout](https://openid.net/specs/openid-connect-rpinitiated-1_0.html#RPLogout)で詳細情報を確認できます。

### CORS 許可オリジン

_CORS（Cross-Origin Resource Sharing）許可オリジン_は、アプリケーションがLogtoサービスへのリクエストを行うことのできる許可されたオリジンのリストです。許可されていないオリジンからは、Logtoサービスへのリクエストはできません。

CORS 許可オリジンのリストは、承認されていないドメインからのログとのサービスへのアクセスを制限し、クロスサイトリクエストフォージェリ（CSRF）攻撃を防止するために使用されます。Logtoでアプリケーションの許可オリジンを指定することにより、サービスは承認されたドメインのみがサービスへのリクエストを行うことを確認できます。

```
許可オリジンのリストには、アプリケーションが提供されるオリジンが含まれている必要があります。これにより、アプリケーションからのリクエストが許可され、承認されていないオリジンからのリクエストがブロックされます。
```

### トークンエンドポイント

_トークンエンドポイント_はOIDCの用語で、OIDCクライアントがOIDCプロバイダからアクセストークン、IDトークン、リフレッシュトークンを取得するためのWeb APIエンドポイントです。

OIDCクライアントがアクセストークンやIDトークンを取得する必要がある場合、トークンエンドポイントに承認グラント（通常は承認コードまたはリフレッシュトークン）を付けたリクエストを送信します。トークンエンドポイントは承認グラントを検証し、グラントが有効である場合はクライアントにアクセストークンまたはIDトークンを発行します。

[トークンエンドポイント](https://openid.net/specs/openid-connect-core-1_0.html#TokenEndpoint)で詳細情報を確認できます。
