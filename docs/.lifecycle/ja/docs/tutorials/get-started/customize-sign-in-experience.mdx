# サインインエクスペリエンスをカスタマイズする

「はじめに」タブで、右側の「カスタマイズ」ボタンをクリックすると、ブラウザはサインインエクスペリエンスタブにリダイレクトします。

## サインインプレビュー

:::tip ご存じですか...
リアルタイムで結果をプレビューできます！左側で更新した内容に関わらず、右側のUIは変更に同期して即座に更新されます。
:::

顧客は、プラットフォーム、言語、外観を含むさまざまな状況でアプリを使用する可能性があります。

自由に変更して、組み合わせの結果を確認してください。

## ブランディング

この領域では、ブランド露出の主要な要素である色、ロゴ、ファビコン、およびカスタムCSSを定義できます。

- **ブランドの色** は、コンポーネントの主要な色（例：ボタンやリンク）に影響します。
- **ダークモードを有効にする** は、Logtoが関連するシステムの設定を検出したときにUIに「ダークな感じ」を持たせるかどうかを制御します。
- **カスタムCSS** を使用すると、CSSコードで全ページのUIをユニークな外観にカスタマイズできます。
- 他のオプションは非常にわかりやすいため、ここではスキップします。

## その他

今は「サインイン方法」をスキップして、「その他」タブに移動しましょう。

### 利用条件

アプリは通常、本番環境で利用する利用規約とプライバシーポリシーを持ち、サインインやアカウントの作成の前にそれを受諾することが必要です。

このプロセスを強制するURLを入力してください。Logtoは `<iframe>` を使用してそれをロードします。

### 言語

このセクションでは、エンドユーザーに表示される言語を指定できます。

ここには特に注意が必要な2つのオプションがあります。1つ目は **自動検出を有効にする** スイッチで、2つ目は **デフォルトの言語を選択する** セレクターです。

デフォルトの設定では、アプリはユーザーのロケール設定を自動的に検出し、Logtoでサポートされている言語の1つに切り替えます。ただし、検出された言語がサポートされていない場合、バックアップとしてデフォルトの言語が使用されます。

この機能を無効にすると、アプリにはデフォルトの言語のみが表示されます。このようにすると、潜在的なユーザーへのアクセスが制限されるため、一般的には推奨されません。

この記事は、アプリの言語設定について感触を掴むための簡単なスタートガイドです。より高度なシナリオや設定推奨については、[ローカライズされた言語](/docs/recipes/customize-sie/localized-language.md) を参照してください。
