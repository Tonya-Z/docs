# コネクタを追加してパスワードレスでサインイン

「はじめる」タブで、右側の「有効にする」ボタンをクリックすると、ブラウザは「コネクタ」にリダイレクトされます。

<ConnectorsIntro />

## Cloud Previewのためのデモコネクタ

Logto Cloudアカウントのオンボーディングプロセス中に、Logtoサービスによって事前に構成されたデモコネクタを選択するオプションがあります。これにより、簡単にパスワードレスでのサインイン体験をお試しいただけます。

「デモ」タグが付いたLogtoデモコネクタには使用制限があることに注意してください。デモEメールまたはSMSコネクタでは100用語まで、デモソーシャルコネクタでは実際にアプリケーションにサインインすることはできません。

テストが完了したら、デモコネクタを削除し、長期的な可用性を確保するために独自のカスタムコネクタを設定する必要があります。

![デモコネクタを削除](./assets/passwordless-delete-demo-connector.png)

## SMSまたはEメールでパスワードレスでサインインを有効にする

Logtoには、次のようなボックスから使用可能な組み込みコネクタがあります。

> SMSコネクタ: Twilio SMS、Aliyun SMS、Tencent SMS
> Eメールコネクタ: AWS Direct Mail、SendGrid Mail Service、Aliyun Direct Mail、SMTP

SMSおよびEメールコネクタの有効化プロセスは似ていますので、便宜上、それらを1つのチュートリアルにまとめています。

![SMSまたはEメールコネクタを追加](./assets/passwordless-add-sms-or-emai-connector.png)

### ステップ1: コネクタを構成してテストする

コネクタを構成するには、「左側のナビゲーションバー」の「コネクタ」タブに移動します。デフォルトでは、このタブは「EメールおよびSMSコネクタ」セクションの下にあります。

SMSまたはEメールコネクタの列で「セットアップ」ボタンをクリックします。次に、モーダルリストから希望のコネクタを選択して、「次へ」をクリックします。

![SMSコネクタを構成](./assets/passwordless-configure-sms-connector.png)

正しくセットアッププロセスをガイドするフルスクリーンページに移動します。以下の手順に従ってください。

1. 左側のREADMEドキュメントを参照し、その後、手順に従います。
2. 右側の必要なフォームに記入して構成を完了し、SMSまたはEメールテンプレートを編集してJSONコードをカスタマイズできます。その後、設定をテストし、あなたのEメールアドレスまたは電話番号にEメールまたはSMSを送信して構成をテストします。
3. 「保存して完了」をクリックして完了します。

![SMSコネクタを構成](./assets/passwordless-configure-sms-connector.png)

### ステップ2：サインイン体験でコネクタを有効にする

SMSまたはEメールコネクタを正常に構成したら、ビジネスニーズに応じてサインアップまたはサインインオプションとして利用できるようにすることができます。

「左側のナビゲーションバー」の「サインインエクスペリエンス」タブに移動し、「サインインメソッド」をクリックします。

1. **サインアップ方法を設定する**: SMSまたはEメールを介したパスワードレスのサインアップを有効にするには、「Eメールアドレス」、「電話番号」、または「Eメールアドレスまたは電話番号」をサインアップ識別子として選択してください。現在のバージョンではサインアップ時にEメールまたは電話の検証が必要です。
2. **サインインメソッドを設定する**: サインアップ識別子は自動的にサインインメソッドとして追加されます。追加のサインイン組み合わせを提供するには、「別の追加」をクリックしてください。
3. 変更を反映させるために、「変更内容を保存」をクリックしてください。

新しいサインインエクスペリエンスをテストするために、「ライブプレビュー」を開いてください。

![SIEでEメールまたはSMSコネクタを有効にする](./assets/passwordless-sie-enable-email-or-sms-connector.png)

## ソーシャルサインインを有効にする

Logtoには、以下のようなボックスから使用可能な多様な組み込みコネクタがあります。

> デフォルトのソーシャルコネクタ: Google、Apple、Facebook、GitHub、Discord、WeChat、Alipay、Kakao、Naver、Azure AD。
> 標準コネクタ: OAuth 2.0、OIDC、SAML。

デフォルトのソーシャルコネクタは、普遍的で人気のあるソーシャルメディアプラットフォームであり、Logtoコンソールに事前インストールされています。同じタイプのコネクタは1つだけ作成できます。

標準コネクタには、一般的に使用されている業界標準の認証および認可目的のためのプロトコルが含まれます。これらのコネクタは通常、独自のソーシャルコネクタを作成したい開発者向けに柔軟性とカスタマイズオプションを提供します。

### ステップ1：ソーシャルコネクタを追加する

ソーシャルコネクタを構成するには、「左側のナビゲーションバー」の「コネクタ」タブに移動し、「ソーシャルコネクタ」タブを選択します。

「ソーシャルコネクタを追加」ボタンをクリックし、最適なコネクタを選択してください。

![ソーシャルコネクタを追加](./assets/passwordless-add-social-connector.png)

フルスクリーンページの指示に従って、左側のREADMEドキュメントを確認し、右側の必要なフォームに記入し、「保存して完了」をクリックして設定を完了します。

![ソーシャルコネクタを構成](./assets/passwordless-configure-social-connector.png)

### ステップ2：サインイン体験でコネクタを有効にする

ソーシャルコネクタを正常に構成したら、サインインエクスペリエンスでそのコネクタをソーシャルサインインボタンとして有効にすることができます。

「左側のナビゲーションバー」の「サインインエクスペリエンス」タブに移動し、「サインインメソッド」をクリックします。

1. **サインアップ方法を選択する**: ソーシャルサインアップのみを提供したい場合は、サインアップ識別子として「該当なし」を選択してください。他のサインアップ識別子を選択する場合、例えばEメールなどを選択すると、ユーザーはサインアップのフロー中にEメールをソーシャルアカウントにリンクする必要があります。
2. **ソーシャルサインインボタンを設定する**: 「別の追加」メニュー内で構成したコネクタをクリックして、ソーシャルサインイン方法としてそれを提供できます。複数のソーシャルコネクタを使用している場合は、ドラッグアンドドロップで並べ替えることができます。
3. 変更を反映させるために、「変更内容を保存」をクリックしてください。

新しいサインインエクスペリエンスをテストするために、「ライブプレビュー」を開いてください。

![SIEでソーシャルコネクタを有効にする](./assets/passwordless-sie-enable-social-connector.png)
