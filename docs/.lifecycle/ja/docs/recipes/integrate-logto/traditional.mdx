# 伝統的なWeb: Logtoと統合

<AppNote type="Traditional Web" />

アプリは、Django、Express、Laravel、NextJSなどのフレームワークを使用してブラウザではなくサーバーサイドで実行される場合、それは従来のWebアプリと呼ばれます。今の段階では、Logtoを手動で統合する必要があります。本記事では、ステップバイステップでの統合方法を説明します。そして、Node.jsのExpressを例に取り上げます。

:::tip

本記事はExpressに限らず、他のフレームワークや他の言語を使用している場合でも、`@logto/js`をその他の言語のコアSDKに置き換え、ステップのいくつかを調整することができます。

:::

## ソースコードを取得

このチュートリアルの最終コードを取得するには、[GitHub](https://github.com/logto-io/express-example)にアクセスできます。

## Expressプロジェクトの開始

`express-generator`を使用すると、簡単にExpressプロジェクトを開始できます。

```bash
mkdir express-logto
cd express-logto
npx express-generator
```

## 依存関係のインストール

デモアプリには4つの依存関係が必要です:

1. **@logto/js**: JavaScript向けのLogtoのコアSDK。
2. **node-fetch**: Node.jsランタイムで`window.fetch`互換の最小コードを提供します。
3. **express-session**: セッションミドルウェア。セッションを使用してユーザートークンを格納します。
4. **js-base64**: 別のBase64トランスコーダーです。

## セッションの使用

ユーザーがサインインすると、トークン（アクセストークン、IDトークン、リフレッシュトークン）とインタラクションデータが取得でき、セッションはそれらを格納する優れた場所です。

前のステップで[express-session](https://github.com/expressjs/session)をインストールしたので、次のコードを追加してセットアップします:

```js
// app.js

const session = require('express-session');

app.use(
  session({
    secret: 'keyboard cat', // 独自のシークレットキーに変更してください
    cookie: { maxAge: 86400 },
  })
);
```

## ユーザーの認証機能の実装

:::tip

次のコードスニペットでは、アプリケーションが `http://localhost:3000` で実行されていると仮定しています。

:::

このステップでは、以下の認証機能を実装する必要があります:

1. `getSignInUrl`: ユーザーがリダイレクトされるLogto認証サーバーの完全なURLを構築して返します。
2. `handleSignIn`: 認証プロセスが完了した後のコールバックURLを解析し、コードクエリパラメーターを取得し、その後トークン（アクセストークン、リフレッシュトークン、IDトークン）を取得してサインインプロセスを完了します。
3. `refreshTokens`: リフレッシュトークンを使用して新しいアクセストークンを交換します。

<GetAppSecret />

```js
// logto.js

const {
  withReservedScopes,
  fetchOidcConfig,
  discoveryPath,
  generateSignInUri,
  verifyAndParseCodeFromCallbackUri,
  fetchTokenByAuthorizationCode,
  fetchTokenByRefreshToken,
} = require('@logto/js');
const fetch = require('node-fetch');
const { randomFillSync, createHash } = require('crypto');
const { fromUint8Array } = require('js-base64');

const config = {
  endpoint: 'https://logto.dev',
  appId: 'foo',
  appSecret: '<your-app-secret-copied-from-console>',
  redirectUri: 'http://localhost:3000/callback', // 必要に応じてアプリの本番アドレスに置き換える必要があります
  scopes: withReservedScopes().split(' '),
};

const requester = (input, init) => {
  const { appId, appSecret } = config;

  return fetch(input, {
    ...init,
    headers: {
      ...init?.headers,
      authorization: `Basic ${Buffer.from(`${appId}:${appSecret}`, 'utf8').toString('base64')}`,
    },
  });
};

const generateRandomString = (length = 64) => {
  return fromUint8Array(randomFillSync(new Uint8Array(length)), true);
};

const generateCodeChallenge = async (codeVerifier) => {
  const encodedCodeVerifier = new TextEncoder().encode(codeVerifier);
  const hash = createHash('sha256');
  hash.update(encodedCodeVerifier);
  const codeChallenge = hash.digest();
  return fromUint8Array(codeChallenge, true);
};

const getOidcConfig = async () => {
  return fetchOidcConfig(new URL(discoveryPath, config.endpoint).toString(), requester);
};

exports.getSignInUrl = async () => {
  const { authorizationEndpoint } = await getOidcConfig();
  const codeVerifier = generateRandomString();
  const codeChallenge = await generateCodeChallenge(codeVerifier);
  const state = generateRandomString();

  const { redirectUri, scopes, appId: clientId } = config;

  const signInUri = generateSignInUri({
    authorizationEndpoint,
    clientId,
    redirectUri: redirectUri,
    codeChallenge,
    state,
    scopes,
  });

  return { redirectUri, codeVerifier, state, signInUri };
};

exports.handleSignIn = async (signInSession, callbackUri) => {
  const { redirectUri, state, codeVerifier } = signInSession;
  const code = verifyAndParseCodeFromCallbackUri(callbackUri, redirectUri, state);

  const { appId: clientId } = config;
  const { tokenEndpoint } = await getOidcConfig();
  const codeTokenResponse = await fetchTokenByAuthorizationCode(
    {
      clientId,
      tokenEndpoint,
      redirectUri,
      codeVerifier,
      code,
    },
    requester
  );

  return codeTokenResponse;
};

exports.refreshTokens = async (refreshToken) => {
  const { appId: clientId, scopes } = config;
  const { tokenEndpoint } = await getOidcConfig();
  const tokenResponse = await fetchTokenByRefreshToken(
    {
      clientId,
      tokenEndpoint,
      refreshToken,
      scopes,
    },
    requester
  );

  return tokenResponse;
};
```

### サインインルートの実装

<SignInNote calling="getSignInUrl()" />

Expressでサインイン用のルートを作成します:

```js
const { getSignInUrl } = require('./logto');

app.get('/sign-in', async (req, res) => {
  const { redirectUri, codeVerifier, state, signInUri } = await getSignInUrl();
  req.session.signIn = { codeVerifier, state, redirectUri };
  res.redirect(signInUri);
});
```

および、コールバックを処理するルート:

```js
app.get('/callback', async (req, res) => {
  if (!req.session.signIn) {
    res.send('不正なリクエストです。');
    return;
  }

  const response = await handleSignIn(
    req.session.signIn,
    `${req.protocol}://${req.get('host')}${req.originalUrl}`
  );
  req.session.tokens = {
    ...response,
    expiresAt: response.expiresIn + Date.now(),
    idToken: decodeIdToken(response.idToken),
  };
  req.session.signIn = null;

  res.redirect('/');
});
```
