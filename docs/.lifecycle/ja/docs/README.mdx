# 💁 はじめに

Logtoは、フロントエンドとバックエンドの両方をカバーし、事前構築されたインフラとエンタープライズグレードのソリューションを備えた包括的なアイデンティティソリューションを提供します。その主な機能は次のとおりです。

- OIDCベースのアイデンティティサービス
- カスタマイズ可能なCSSを使用したユーザーフレンドリーなサインイン体験
- ソーシャル、メール、電話番号、ユーザー名、パスワードレス認証を含むさまざまなサインインオプション
- 使用可能な管理APIは、認証プロバイダーとして機能することができます
- アプリを迅速にLogtoと統合できるSDK
- コミュニティの貢献でスケーラブルになり、SMAL、OAuth、およびOIDCプロトコルでカスタマイズできる柔軟なコネクタ
- 多様なユースケースのためにリソースを制御するためのスケーラブルなロール認可のためのRBAC
- ユーザー管理と監査ログは、ユーザーのアイデンティティに関する情報を把握し、セキュリティをトラックするのに役立ちます。

私たちの問題領域は、一般的に"[customer identity access management](https://en.wikipedia.org/wiki/Customer_identity_access_management)"（CIAM）または"customer identity solution"と呼ばれます。これらの用語は最もエキサイティングなものではないかもしれませんが、私たちの作業の焦点を正確に表現しています。

## 基本的な概念

Logtoの動作をよりよく理解するために、次の4つのパーツに分けることができます：管理コンソール、サインイン体験、コアサービス、およびSDK。

- **管理コンソール** は、リソースとサインイン体験を管理することができるWebアプリです。
- **サインイン体験** は、サインインやアカウントの作成など、エンドユーザーが対話するユーザーインターフェースです。
- **コアサービス** は、Logtoの基盤であり、他のすべての機能をサポートするために必要なAPIを提供します。
- **SDK** は、アプリケーションとLogtoの間の橋渡しとなります。
